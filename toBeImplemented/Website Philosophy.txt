PRD Booking Venue Website
Halaman 1: Auth (Login & Register) Tujuan: Memastikan proses autentikasi pengguna berjalan
aman dan efisien sebelum mengakses seluruh fitur website.
Fitur Utama: 1. Login - Input: username dan password - Validasi akun terdaftar - Autentikasi role:
user 2. Register - Input: username, password, konfirmasi password - Validasi username unik -
Setelah berhasil register ® dialihkan ke halaman login 3. Logout - Tombol logout pada navbar
kanan atas - Menghapus sesi pengguna dan kembali ke halaman login
Alur User: 1. User membuka website ® langsung diarahkan ke halaman login. 2. Jika belum punya
akun ® klik “Register” ® isi form ® berhasil ® kembali ke login. 3. Jika login sukses ® diarahkan
ke halaman Home (Landing Page).
Integrasi: Home, Catalog, Wishlist, Booking, Review
Template: Login.html, Register.html
------------------------------------------------------------
Halaman 2: Home (Landing Page) Tujuan: Menjadi halaman utama setelah login yang
menampilkan venue populer dan menyediakan fitur pencarian cepat.
Fitur Utama: 1. Navbar - Kiri: Logo - Tengah: Home, Catalog, Wishlist - Kanan: Username, ikon
profil, dan tombol logout 2. Hero Section - Gambar utama yang relevan dengan tema website
(venue olahraga) - Filter pencarian: Lokasi (kota), Kategori, Harga 3. Most Popular Venues -
Menampilkan 3 venue dengan jumlah booking terbanyak - Aksi: - Klik Love ® tambah ke wishlist -
Klik View Product ® buka halaman detail produk
Alur User: 1. User masuk ke Home ® dapat melakukan filtering (berdasarkan satu atau lebih filter).
2. Hasil filter menampilkan venue yang sesuai. 3. User dapat menekan Love untuk menyimpan
venue ke wishlist atau View Product untuk melihat detail.
Integrasi: Catalog, Wishlist, Product Detail
Template: Mainpage.html
------------------------------------------------------------
Halaman 3: Catalog Tujuan: Menampilkan semua venue yang tersedia dan memungkinkan
pengguna melakukan filtering yang lebih spesifik.
Fitur Utama: 1. Daftar Venue Lengkap - Ditampilkan dalam bentuk katalog grid 2. Filter -
Berdasarkan lokasi (kota), kategori, dan harga 3. Aksi di Setiap Kartu Venue - Klik Love ® simpan
ke wishlist - Klik View Product ® buka halaman detail
Alur User: 1. User klik “Catalog” di navbar ® diarahkan ke halaman katalog. 2. User dapat memilih
filter (misal: kota = Jogja, kategori = voli, harga tertentu). 3. Hasil katalog menyesuaikan dengan
filter. 4. User dapat melakukan aksi yang sama seperti di halaman Home.
Integrasi: Home, Wishlist, Product Detail
Template: Product Catalog.html
------------------------------------------------------------
Halaman 4: Product Detail Tujuan: Memberikan informasi detail venue dan memungkinkan user
melakukan booking serta melihat review.
Fitur Utama: 1. Informasi Venue Lengkap - Deskripsi, harga, fasilitas, foto, lokasi 2. Fitur Interaktif -
Klik Love ® simpan ke wishlist - Booking Here ® tampilkan kalender tanggal dan jam tersedia -
Add Review ® user bisa menulis ulasan 3. Review Section - Menampilkan review dari user lain -
Form untuk menambahkan review baru
Alur User: 1. User klik “View Product” ® masuk ke halaman detail. 2. User dapat memilih
tanggal/jam booking, memilih add-on opsional, lalu klik “Book”. 3. Setelah booking, diarahkan ke
halaman Payment.
Integrasi: Booking, Wishlist, Review
Template: Product Details.html
------------------------------------------------------------
Halaman 5: Booking & Payment Tujuan: Memfasilitasi user dalam melakukan pemesanan venue
dan pembayaran secara online.
Fitur Utama: 1. Booking System - Pilih tanggal dan jam yang tersedia - Pilih add-on tambahan
(opsional) - Tampilkan ringkasan di “Your Picks” 2. Payment Page - Tampilkan invoice: DP
(Rp10.000) + total harga sesuai venue dan add-on - Metode pembayaran: QRIS / GoPay - Status:
Waiting for confirmation ® Confirm ® Selesai (redirect ke landing page “mata senyum”)
Alur User: 1. User klik “Booking Here” di halaman detail venue. 2. Pilih tanggal, jam, dan add-on ®
klik “Book”. 3. Masuk ke Payment Page ® pilih metode pembayaran ® konfirmasi.
Integrasi: Auth, Product Detail, Payment Gateway
Remark: berupa modal pada product details.html
------------------------------------------------------------
Halaman 6: Wishlist Tujuan: Menyimpan semua venue yang disukai oleh user agar mudah diakses
kembali.
Fitur Utama: 1. Daftar Wishlist - Menampilkan semua venue yang pernah diklik “Love” 2. Aksi - Klik
View Product ® buka halaman detail - Klik Unlove ® hapus dari wishlist
Alur User: 1. User klik “Wishlist” di navbar ® diarahkan ke halaman wishlist. 2. User dapat melihat
seluruh venue favorit, menghapus dari daftar, atau masuk ke detail venue.
Integrasi: Home, Catalog, Product Detail, Booking
Remark: buat halaman profile page sendiri
------------------------------------------------------------
Halaman 7: Review Tujuan: Menjadi wadah bagi user untuk berbagi pengalaman dan membantu
user lain menilai venue.
Fitur Utama: 1. Tambah Review: rating dan komentar 2. Lihat Review Lain: semua review tampil di
detail venue 3. Review terhubung ke username user
Alur User: 1. User masuk ke detail venue ® buka bagian review. 2. User bisa menulis review baru
dan melihat review pengguna lain.
Integrasi: Auth, Product Detail, Booking
Template: Product Details.html
